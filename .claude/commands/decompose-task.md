# 문제를 Phase별로 분해하기

당신은 문제를 체계적으로 분해하는 전문가입니다.

## 입력된 문제
$ARGUMENT

## 작업 지침

1. **프로젝트 구조 파악**
   - 관련 파일 검색 (Glob/Grep)
   - 기존 코드 패턴 확인
   - 의존성 분석

2. **Phase 분해 기준**
   - 각 Phase는 1-3시간 내 완료 가능
   - 독립적으로 검증 가능
   - 명확한 완료 조건
   - SOLID 원칙 준수
   - 각 Phase마다 lint + build 체크

3. **SPEC.md 형식으로 출력**

```markdown
# [문제명] 개선 사양서

## 프로젝트 개요
[간단한 설명]

---

## Phase 1: [제목]

### 📋 이번 단계 개선 사항 오버뷰
[목적 설명]

### ✅ 수정/개선해야 하는 기능
- [ ] 작업 1 (예상: 30분)
- [ ] 작업 2 (예상: 1시간)

### ✅ 작업 완료 후 검증/확인 사항
- [ ] TypeScript 컴파일 에러 없는지 확인
- [ ] ESLint 에러 없는지 확인
- [ ] 기능 작동 확인

---

## Phase 2: [제목]
...

---

## 전체 Phase 체크리스트
- [ ] Phase 1: [제목]
- [ ] Phase 2: [제목]
```

4. **기술 스택**
   - Next.js 15 App Router
   - TypeScript
   - Tailwind CSS
   - pnpm
   - 변수명: camelCase

이제 위 문제를 분석하고 Phase별로 분해해주세요.
